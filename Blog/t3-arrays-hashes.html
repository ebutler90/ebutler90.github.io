Covering: Arrays vs. Hashes

Sunday, January 11, 2015

Hello again! It's time to talk about Arrays and Hashes. At first glance, you'd think they were basically the same thing. When I was first reading about them, I was having some trouble differentiating them, and really understanding when an array was appropriate and when a hash was appropriate. But, there are some key differences, which I will break down in the following paragraphs.

Before we start in on differences, lets explain how they are the same. Both Arrays and Hashes are what we call "indexed collections." They both store objects, that are essentially arranged in lists, and all of the objects can be called upon via a key. Both Hashes and Arrays can be added to or deleted from. Most importantly howeverm both hashes and arrays can hold any types of data, whether they be integers, floats, strings, etc.

The main difference between arrays and hashes is how you can access the individual objects. With an array, the key will always be an integer. (The key will start at 0 which relates to the first object in the array, and increase from there to match the remainder of the objects in the array.) Hashes on the other hand can use any object as a key to the objects that they hold. This makes arrays more efficient, since there is a uniform call system. But hashes are a more flexible option, because they allow you to map one value to another. For example:

array = [ "cat", "dog", "bird"]

To call the objects, you would have to use the following:

puts array[0] (This would call "cat")
puts array[1] (This would call "dog")
puts array[2] (This would call "bird")

But with a hash, things are a little different. You can assign any value (it doesnt have to be 0..100+):
hash = {"whiskers" => "cat", "rufus" => "dog", "tweety" => "bird"}

To call the objects, you can use the word values:

puts hash["whiskers"] (This would call "cat")
puts hash["rufus"] (This would call "dog")
puts hash["tweety"] (This would call "bird")
